Propeller

Propeller is a WebAssembly (Wasm) orchestrator and rule engine built atop SuperMQ. It enables dynamic, event-driven logic execution across distributed IoT and edge environments, allowing developers to deploy and manage Wasm modules seamlessly.

Download Propeller Read the Docs >

What is Propeller

Propeller serves as the execution engine for Abstract Machines, enabling real-time, event-driven processing through WebAssembly modules. It allows developers to write portable, secure, and high-performance logic that can be deployed across diverse environments.

By leveraging SuperMQ's messaging capabilities, Propeller ensures that Wasm modules can react to events, process data, and enforce rules efficiently, making it ideal for complex IoT and edge computing scenarios.

Propeller is open-source and designed for extensibility, supporting various use cases from simple data transformations to complex decision-making workflows.

Propeller Features

Wasm Module Orchestration

Deploy and manage WebAssembly modules dynamically, enabling portable logic execution across devices and services.

Event-Driven Execution

Trigger Wasm execution in response to SuperMQ events, enabling responsive and low-latency edge intelligence.

Seamless SuperMQ Integration

Built to work hand-in-hand with SuperMQ for ultra-fast message-driven orchestration.

Sandboxed & Secure

Propeller runs each Wasm module in isolation, offering strong safety and deterministic execution guarantees.

Observability

Integrated metrics, logs, and traces for performance monitoring and debugging across Wasm executions.

Open-Source & Extensible

Licensed under Apache 2.0. Extend Propeller to meet your needs, or contribute to the growing Wasm community.
